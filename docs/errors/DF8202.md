# DF8202: Terminal Session Does Not Accept Input

## Message

> Terminal session "`{id}`" does not accept input

## Cause

`hub:terminals:write` or `hub:terminals:resize` was called for a session that
is output-only. Sessions started with `ctx.terminals.startChildProcess()` are
pipe-backed and read-only; only PTY sessions from
`ctx.terminals.startPtySession()` accept keystrokes and resize.

## Fix

- Spawn the session via `ctx.terminals.startPtySession(executeOptions, terminal)`
  to get an interactive, writable PTY.
- If you only need to stream output, keep using `startChildProcess()` and don't
  wire input to it.

## Source

- [`packages/hub/src/node/rpc-builtins.ts`](https://github.com/devframes/devframe/blob/main/packages/hub/src/node/rpc-builtins.ts) — the `hub:terminals:write` / `hub:terminals:resize` handlers throw this when the resolved session has no `write` handle.
