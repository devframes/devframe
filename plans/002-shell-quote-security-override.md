## Why this matters

`shell-quote@1.8.3` has a **critical** advisory (GHSA-w7jw-789q-3m8p: `quote()`
mishandles certain inputs, an argument-injection boundary), patched in `1.8.4`.
It reaches devframe via `launch-editor` and — critically — `packages/devframe/tsdown.config.ts`
lists `shell-quote` in `onlyBundle`, so the vulnerable code is **inlined into
the published `devframe` `dist/`** and ships to every consumer. The editor-launch
recipe builds a command from a file path through `shell-quote`. A pinned override
plus a rebuild removes the flagged code from the shipped bundle with no source
changes.

## Steps

### Step 1: Add the override

In `pnpm-workspace.yaml`, add to the existing `overrides` block:

```yaml
overrides:
  chokidar: ^5.0.0
  crossws: ^0.4.8
  semver: ^7.8.1
  shell-quote: ^1.8.4
```

### Step 2: Relock

Run `pnpm install`.

**Verify**: `grep -n "shell-quote@1.8" pnpm-lock.yaml` shows only `1.8.4` (or
higher) — no remaining `shell-quote@1.8.3`.

### Step 3: Confirm the advisory is gone

**Verify**: `pnpm audit --prod` no longer lists GHSA-w7jw-789q-3m8p / the
`shell-quote` critical. (Other pre-existing advisories under `examples`/`docs`
may still appear — those are out of scope, handled in plan 028.)

### Step 4: Rebuild and confirm the shipped bundle is patched

Rebuild devframe so `dist` reflects the new version:
`pnpm --filter devframe build`.

**Verify**: the bundled shell-quote is 1.8.4+. Inspect the built output for the
version marker, e.g. check the installed dependency version the bundle was built
from: `pnpm --filter devframe why shell-quote` (or `pnpm ls shell-quote -r`)
resolves to ≥1.8.4.
